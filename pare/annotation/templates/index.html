Welcome to the PARE Annotation Study

Thank you for participating in this study. You'll be evaluating proposals made by an AI assistant on a mobile phone.

Your Task

You will see a timeline of interactions between a user and their AI assistant on a simulated mobile phone. The user performs actions like opening apps, reading emails, checking their calendar, and browsing messages. At some point, the AI assistant notices an opportunity to help and makes a proposal -- a suggestion to take action on the user's behalf.

Your job: If you were this user, would you want the assistant to do what it's proposing?

Think about:

  • Does the assistant understand the situation correctly?
  • Is the proposed action actually what the user would want right now?
  • Does the user have enough context to trust this proposal?
  • Is the timing appropriate?

Your Three Options

ACCEPT

The proposal is helpful and appropriate

The assistant correctly understood the user's situation. The proposed action makes sense, the information is accurate, and the user would benefit from the assistant doing this. The user has seen enough context to trust the proposal.

REJECT

The proposal is wrong, unhelpful, or inappropriate

The assistant misunderstood the situation, the proposed action is incorrect, the timing is bad, or the assistant is being too pushy (e.g., trying to complete a purchase without the user actively wanting to buy). Reject even if the proposal seems "close" -- if it's not what the user would actually want, reject it.

I NEED MORE INFO

You can't decide without checking something first

The proposal might be good, but the user would need to verify something before committing. For example: they'd want to read a full email before replying to it, check their calendar before accepting a meeting, or review their shopping cart before checking out. You'll be asked to briefly describe what information you'd need. Important: Only choose this when there's a genuine information gap -- if the user has already seen the relevant information, make a decision (accept or reject) rather than defaulting to "I need more info."

Starting State

The user always starts on the home screen of their mobile phone with no apps open. They navigate by opening apps and performing actions. You can track which app they're currently in by following their actions from top to bottom.

Important

Put yourself in the user's shoes. The key question is: what does the user know at the moment the proposal arrives? A proposal that's objectively correct can still deserve "I need more info" if the user hasn't seen the relevant information yet. Conversely, don't choose "I need more info" if the user has already verified the details -- make a decision. There are no trick questions. Trust your instincts.
